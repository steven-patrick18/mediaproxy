# Installing mediaproxy

Three install paths. Most are one-click; only the control-plane needs a real install.

## ЁЯЯв Path 1 тАФ Add a new SipProxy or MediaNode

From the panel: **Infrastructure тЖТ Nodes тЖТ Add node** тЖТ fill in details тЖТ **Provision via SSH**.

The provisioner installs everything on the new host:

- Ubuntu nftables + at (firewall auto-apply)
- **sip_proxy role:** Kamailio 5.8 from the kamailio.org repo + systemd NPROC/NOFILE limits drop-in (prevents libcurl resolver thread starvation under load)
- **media role:** Ubuntu's `rtpengine` package + enables `rtpengine-daemon.service` + iptables-persistent + an ACCEPT for UDP/2223 above the rtpengine kernel hook (so kamailio's NG control packets reach userspace)
- Agent binary from `https://<your-domain>/agent-binary`
- systemd unit + a pre-seeded `/etc/node-agent/config.yaml`

Then click **Apply firewall** тАФ the control plane renders the right nftables ruleset for the role (sip_proxy gets the carrier+dialer allowlist; media gets the RTP port range + UDP/2223 from registered sip_proxy IPs).

### Split-host topology (rtpengine on media node, Kamailio on sip_proxy node)

After adding both nodes, on the SipProxy edit `/etc/node-agent/config.yaml` and uncomment:

```yaml
rtpengine_sock: "udp:<MediaNode-IP>:2223"
```

Restart the agent: `systemctl restart node-agent`. The agent will regenerate `kamailio.cfg` with `rtpengine_sock` pointing at the media node and Kamailio will hot-reload.

## ЁЯЯб Path 2 тАФ Fresh control plane (Base app)

On a clean Ubuntu 24.04 VPS:

```bash
# Clone the repo
git clone git@github.com:steven-patrick18/mediaproxy.git /opt/mediaproxy
cd /opt/mediaproxy

# One-shot installer
sudo ./scripts/install.sh \
    --domain mediaproxy.example.com \
    --admin-email you@example.com
    # --admin-password optional; auto-generated and printed if omitted
    # --skip-homer to skip the HOMER SIP-ladder stack
    # --skip-tls to skip certbot (e.g. for local dev)
```

The script is **idempotent**. Re-running it picks up where it left off тАФ safe to interrupt and restart.

What it does:

1. `apt-get` installs postgres, redis, nginx, go (1.23 from the official tarball if system go is old), certbot, docker, build-essential
2. Creates the `mediaproxy` postgres role + database (reuses existing password if `.env` is already present тАФ never silently rotates secrets)
3. Generates `.env` with a strong DB password and 32-byte JWT secret (mode 0600)
4. Installs `golang-migrate`, runs `migrate -path migrations up`
5. Builds `bin/baseapp`, `bin/seedadmin`, `bin/node-agent`, `bin/node-agent-linux-amd64`
6. Seeds the admin user (skipped if email already exists)
7. Installs `/etc/systemd/system/mediaproxy-baseapp.service` and starts it
8. Drops the nginx site template with your domain substituted, gets a Let's Encrypt cert via certbot, reloads nginx
9. Brings up the HOMER docker-compose stack at `/opt/homer/` (unless `--skip-homer`)

Total time: ~5 minutes on a 2-core VPS.

When it finishes you get a summary printed:

```
  Panel:           https://YOURDOMAIN
  Admin login:     you@example.com
  Admin password:  <auto-generated or yours>
  HOMER UI:        https://YOURDOMAIN/homer/  (admin / sipcapture)
```

**Save the admin password.** It's printed once and not stored anywhere except hashed in the DB.

## ЁЯФ┤ Path 3 тАФ Disaster recovery / migrating control-plane

What's at risk if the control-plane host dies:

- **Configuration** (clients, carriers, IPs, assignments, firewall rules) тАФ in PostgreSQL
- **CDR history** тАФ in PostgreSQL
- **Active calls in flight** тАФ not durable, lost on baseapp restart > a few seconds (agents will re-register, in-flight calls fail through, dialer retries)
- **Auth** тАФ JWT secrets in `.env`; if lost, every admin must re-login but their accounts remain
- **HOMER SIP captures** тАФ in the homer-db container's volume

What to back up nightly:

```bash
# 1. Postgres dump
pg_dump -Fc mediaproxy > /backups/mediaproxy-$(date +%F).dump

# 2. .env files (have the DB password + JWT secret)
cp /opt/mediaproxy/.env /backups/mediaproxy.env-$(date +%F)
cp /opt/homer/.env       /backups/homer.env-$(date +%F)

# 3. HOMER DB
docker exec homer-db pg_dump -U homer -Fc homer_data > /backups/homer-data-$(date +%F).dump
```

Push these offsite тАФ the repo ships `deploy/backups/offsite.example.sh` as a starting template.

To restore on a new VPS:

1. Follow **Path 2** but pass `--skip-homer` (do HOMER restore separately)
2. After `install.sh` finishes, stop baseapp: `systemctl stop mediaproxy-baseapp`
3. Restore `.env`: `cp /backups/mediaproxy.env-* /opt/mediaproxy/.env`
4. Drop + restore the DB:
   ```bash
   sudo -u postgres psql -c "DROP DATABASE mediaproxy"
   sudo -u postgres psql -c "CREATE DATABASE mediaproxy OWNER mediaproxy"
   pg_restore -d mediaproxy -U mediaproxy -h 127.0.0.1 /backups/mediaproxy-*.dump
   ```
5. Start baseapp: `systemctl start mediaproxy-baseapp`
6. Agents on existing nodes will reconnect within ~10s

DNS: if you're keeping the same domain, no other change needed. If you're moving to a new domain you'll need to update `agent_token` тЖТ agents must be re-provisioned with the new URL.

## Build dependencies (for `make build`)

- Go 1.22+ (the installer fetches 1.23 if missing)
- Node 18+ (for `make ui-build`)
- `make`, `git`, `gcc`, `libc-dev` (`build-essential` covers these)
- Postgres + Redis at runtime
- Docker (for HOMER only)

## Make targets summary

| Target                    | What it does                                                |
|---------------------------|-------------------------------------------------------------|
| `make build`              | host-platform: baseapp + seedadmin + node-agent             |
| `make build-agent-static` | linux/amd64 static binary served from `/agent-binary`       |
| `make migrate-up`         | apply schema migrations                                     |
| `make seed-admin`         | interactive admin user seed (use `seedadmin` flags directly for non-interactive) |
| `make ui-build`           | build the React panel (web/) into `web/dist/`               |
| `make tidy`               | `go mod tidy`                                               |
| `make clean`              | remove `bin/`                                               |
